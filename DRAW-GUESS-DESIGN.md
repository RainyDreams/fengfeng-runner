# 你画我猜 - 完整设计方案

## 一、游戏概述

你画我猜是一款多人实时互动绘画猜词游戏。一名玩家根据题目在画布上作画，其他玩家通过观察画作猜出答案。

- **人数**: 2-6人/房间
- **模式**: 轮流作画，其余人猜
- **平台**: Web端（PC + 移动端自适应）
- **通信**: WebSocket 实时同步

---

## 二、核心玩法

### 2.1 游戏流程

```
创建/加入房间 → 选择难度 → 等待玩家 → 开始游戏
                                          ↓
                        ┌─────────────────────────────┐
                        │  轮流作画（60-90秒/回合）     │
                        │  当前画者选择题目（3选1）      │
                        │  其他玩家实时猜词              │
                        │  猜对即得分，按时间递减        │
                        └─────────────────────────────┘
                                          ↓
                              所有轮次结束 → 排名展示
                                          ↓
                                    再来一局 / 退出
```

### 2.2 回合机制

| 参数 | 设置 |
|------|------|
| 每局轮数 | 房主可选：3/5/7 轮 |
| 每回合时间 | 60/75/90 秒可选 |
| 选题时间 | 10 秒（3选1） |
| 结果展示 | 3 秒 |
| 画者无人猜对 | 画者 0 分，下一轮继续 |

### 2.3 计分规则

| 事件 | 得分 |
|------|------|
| 画者：有人猜对 | +20 |
| 猜者：前 10 秒猜对 | +50 |
| 猜者：10-30 秒猜对 | +30 |
| 猜者：30 秒后猜对 | +10 |
| 猜者：连续猜对 3 题 | +30 额外奖励 |
| 画者：无人猜对 | 0 |

### 2.4 题库设计

**难度分级：**

| 难度 | 示例 | 画者加分 | 猜者基础分 |
|------|------|---------|-----------|
| 简单 | 苹果、太阳、猫、房子 | ×1 | ×1 |
| 中等 | 过山车、消防员、火锅 | ×1.5 | ×1.5 |
| 困难 | 霸王龙、望远镜、钢琴 | ×2 | ×2 |

**题库分类（建议）：**

```
动物: 猫、狗、熊猫、长颈鹿、鲨鱼...
食物: 汉堡、寿司、披萨、冰淇淋、火锅...
人物: 超人、孙悟空、圣诞老人...
物品: 手机、自行车、雨伞、钢琴...
地点: 长城、埃菲尔铁塔、太空站...
成语: 画蛇添足、守株待兔、掩耳盗铃...
电影: 泰坦尼克号、哈利波特、复仇者联盟...
```

## 六、技术架构

### 6.1 前后端通信

```
客户端 ←──WebSocket──→ 服务端

事件类型:
├── 客户端 → 服务端
│   ├── draw        (画笔坐标)
│   ├── guess       (猜词内容)
│   ├── join-room   (加入房间)
│   ├── create-room (创建房间)
│   └── ready       (准备就绪)
│
└── 服务端 → 客户端
    ├── draw        (广播画笔)
    ├── guess-result(猜词结果)
    ├── turn-start  (回合开始)
    ├── turn-end    (回合结束)
    ├── player-join (玩家加入)
    ├── player-leave(玩家离开)
    └── game-over   (游戏结束)
```

### 6.2 画笔数据格式

```json
{
    "type": "draw",
    "roomId": "ABC123",
    "data": {
        "tool": "pen",
        "color": "#000000",
        "size": 3,
        "points": [
            [120, 80],
            [122, 82],
            [125, 85]
        ]
    }
}
```

**优化：**
- 只发送关键点（间隔 > 2px 才记录）
- 客户端用贝塞尔曲线插值平滑
- 每 50ms 批量发送一次（非实时逐点）

- **端口**: 4000
- **内存限制**: 200MB (PM2 --max-memory-restart 200M)
- **Nginx**: 需配置 WebSocket 支持

### 8.2 Nginx 配置

```nginx

    server_name draw.linkbrain.top;

    location / {
        proxy_pass http://127.0.0.1:4000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

## 九、性能优化

### 9.1 画笔传输优化

- 坐标压缩：只发关键点，客户端插值
- 批量发送：每 50ms 合并一次
- 二进制传输：用 ArrayBuffer 替代 JSON

### 9.2 内存优化

- 房间数据存内存，不持久化
- 玩家离线 30 秒自动清理
- 每局结束清理画布数据
- 限制最大房间数（50 个）

### 9.3 带宽优化

- 画笔坐标用相对值（相对于画布尺寸）
- 静态资源 gzip 压缩
- 图片资源 CDN 加速
